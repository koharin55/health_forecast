# iOSショートカット連携ガイド

PreCareのAPIを使って、iOSの「ショートカット」アプリからヘルスケアデータを自動送信する方法を説明します。

## 前提条件

- iOS 16以降（iOS 26推奨）
- PreCareアカウント（APIトークン生成済み）
- iPhoneの「ヘルスケア」アプリにデータが記録されていること

## APIトークンの取得

1. PreCareにログイン
2. マイページ → 「API連携」セクションを開く
3. 「トークンを生成」をタップ
4. 表示されたトークンをコピー（**このトークンは一度しか表示されません**）

## サンプルショートカットのインポート

管理者がiCloudリンクを用意している場合は、リンクからショートカットをインポートできます。
インポート時に以下の質問に回答してください:

1. **APIトークン**: マイページで生成したトークン
2. **サーバーURL**: PreCareのURL（デフォルト値が設定されている場合はそのままでOK）

## 手動でショートカットを作成する

### Step 1: 新規ショートカット作成

「ショートカット」アプリを開き、右上の「+」から新しいショートカットを作成します。

### Step 2: バイタルデータの取得

「ヘルスケアサンプルを検索」アクションを6回追加し、それぞれ以下のように設定します:

| 種類 | グループ化 | ソート順 | 制限 |
|------|-----------|---------|------|
| 体重 | なし | 開始日（新しい順） | 1件 |
| 歩数 | なし | 開始日（新しい順） | 1件 |
| 心拍数 | なし | 開始日（新しい順） | 1件 |
| 最高血圧 | なし | 開始日（新しい順） | 1件 |
| 最低血圧 | なし | 開始日（新しい順） | 1件 |
| 体温 | なし | 開始日（新しい順） | 1件 |

> **ヒント**: 各アクションの出力（マジック変数）をタップ → 「名前を変更」でわかりやすい名前を付けると、後の手順で識別しやすくなります。

### Step 3: 睡眠・運動データの取得

#### 3-1. 睡眠データ（2アクション）

iOS 16以降のApple Watchは睡眠を「コア・深い・REM・起床中（Awake）・就寝中（InBed）」のステージで記録します。「ヘルスケアサンプルを検索」でフィルタリングし、「統計を計算」で合計することで、従来7アクション必要だった処理を2アクションに簡略化できます。

**アクション①: ヘルスケアサンプルを検索**

| 設定項目 | 値 |
|---------|-----|
| 種類 | 睡眠 |
| フィルター（2条件 / すべての条件に一致） | 睡眠ステージ 次と等しくない 起床中 |
| | 睡眠ステージ 次と等しくない 就寝中 |
| 日付 | 昨日 |

**アクション②: 統計を計算**

| 設定項目 | 値 |
|---------|-----|
| 入力 | ①の検索結果 →「期間」を選択 |
| 操作 | 合計 |
| 出力ラベル | sleep_seconds |

> **「就寝中」も除外する理由**: Apple Watchの詳細ステージ記録では、睡眠期間全体をカバーする「就寝中（InBed）」サンプルが、ステージ別サンプル（コア/深い/REM/起床中）とは別に作成されます。「起床中」だけを除外すると就寝中サンプルも合算され、睡眠時間が二重カウントされます。

> **補足**: Apple Watchで睡眠追跡していない場合はこの手順を省略できます。

#### 3-2. 運動時間（2アクション）

Apple Watchのアクティビティリングの「運動」値（Apple 運動時間）を取得します。睡眠と同様に2アクションで本日の合計運動時間（分単位）を取得します。

**アクション①: ヘルスケアサンプルを検索**

| 設定項目 | 値 |
|---------|-----|
| 種類 | Apple 運動時間 |
| 日付 | 今日 |

**アクション②: 統計を計算**

| 設定項目 | 値 |
|---------|-----|
| 入力 | ①の検索結果 |
| 操作 | 合計 |
| 出力ラベル | exercise_minutes |

> **補足**: Apple 運動時間の「統計を計算」で返る値は分単位の整数です。睡眠（期間→秒）と異なり秒変換は不要で、そのまま `exercise_minutes` として送信します。Apple Watchを使用していない場合はこの手順を省略できます。

### Step 4: 手入力データのアクション追加

「入力を要求」アクションを2つ追加します:

- **体調スコア**: プロンプト「今日の体調は？(1:とても悪い〜5:とても良い)」、入力タイプ「数字」
- **メモ**: プロンプト「メモ（任意）」、入力タイプ「テキスト」

### Step 5: APIリクエストの送信

「URLの内容を取得」アクションを追加し、以下のように設定します:

- **URL**: `https://your-domain.com/api/v1/health_records`
- **方法**: POST
- **ヘッダー**:
  - `Authorization`: `Bearer あなたのAPIトークン`
- **本文**: JSON

JSONのキーと値:

| キー | 値（アクションの出力結果） | 備考 |
|------|------------------|------|
| `recorded_at` | 現在の日付（yyyy-MM-dd形式） | |
| `weight` | 体重データの値 | サーバーが小数第1位へ自動丸め |
| `steps` | 歩数データの値 | |
| `heart_rate` | 心拍数データの値 | |
| `systolic_pressure` | 最高血圧データの値 | |
| `diastolic_pressure` | 最低血圧データの値 | |
| `body_temperature` | 体温データの値 | |
| `sleep_seconds` | sleep_seconds（Step 3-1で取得） | コア+深い+REMの合計秒数 |
| `exercise_minutes` | exercise_minutes（Step 3-2で取得） | Apple 運動時間（分） |
| `mood` | 体調スコア入力の値 | |
| `notes` | メモ入力の値 | |

### Step 6: 通知の表示

「通知を表示」アクションを追加し、送信完了を確認できるようにします。

### Step 7: オートメーション設定（任意）

毎日自動で実行するには、「オートメーション」タブで:

1. 「+」→「時刻」を選択
2. 実行時刻を設定（例: 毎日 22:00）
3. 「すぐに実行」を選択
4. 作成したショートカットを選択

> **注意**: 「入力を要求」アクションを含む場合、自動実行時にはプロンプトが表示されます。完全自動化したい場合はStep 4を省略してください。

## APIリファレンス

### エンドポイント

```
POST /api/v1/health_records
```

### 認証

Bearer Token認証:

```
Authorization: Bearer <APIトークン>
```

### パラメータ

| パラメータ | 型 | 必須 | 説明 |
|-----------|------|------|------|
| `recorded_at` | date | **必須** | 記録日（yyyy-MM-dd形式） |
| `weight` | decimal | 任意 | 体重（kg）。小数第1位へ自動丸め |
| `mood` | integer | 任意 | 体調スコア（1-5） |
| `sleep_seconds` | integer | 任意 | 睡眠秒数（コア+深い+REM）。サーバーで分へ変換して保存 |
| `exercise_seconds` | integer | 任意 | 運動秒数。サーバーで分へ変換して保存（exercise_minutesより優先） |
| `exercise_minutes` | integer | 任意 | 運動時間（分）。Apple 運動時間の値をそのまま送信 |
| `steps` | integer | 任意 | 歩数 |
| `heart_rate` | integer | 任意 | 心拍数（bpm） |
| `systolic_pressure` | integer | 任意 | 最高血圧（mmHg） |
| `diastolic_pressure` | integer | 任意 | 最低血圧（mmHg） |
| `body_temperature` | decimal | 任意 | 体温 |
| `notes` | text | 任意 | メモ |

### curlコマンド例

```bash
curl -X POST https://your-domain.com/api/v1/health_records \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "recorded_at": "2026-03-05",
    "weight": 65.53,
    "mood": 4,
    "sleep_seconds": 27000,
    "exercise_minutes": 30,
    "steps": 8000,
    "heart_rate": 72,
    "systolic_pressure": 120,
    "diastolic_pressure": 80,
    "body_temperature": 36.5,
    "notes": "今日は調子が良い"
  }'
```

> **補足**: `weight: 65.53` はサーバーで `65.5` に丸められます。`sleep_seconds: 27000`（7時間30分）はサーバーで `sleep_minutes: 450` に変換されます。`exercise_minutes: 30` はそのまま保存されます。

### レスポンス例

**成功時 (201 Created)**:
```json
{
  "status": "success",
  "health_record": {
    "id": 1,
    "recorded_at": "2026-02-20",
    "weight": "65.5",
    "mood": 4,
    ...
  }
}
```

**エラー時 (422 Unprocessable Entity)**:
```json
{
  "status": "error",
  "errors": ["記録日を入力してください"]
}
```

## トラブルシューティング

| エラー | 原因 | 対処法 |
|--------|------|--------|
| 401 Unauthorized | APIトークンが無効 | マイページでトークンを確認・再生成 |
| 422 Unprocessable Entity | バリデーションエラー | recorded_atの日付形式やmoodの範囲を確認 |
| ヘルスケアデータ取得不可 | アクセス権限が未許可 | 設定 → プライバシー → ヘルスケア → ショートカットで許可 |
| 同日の重複送信 | — | 同じ日付の記録は自動的に更新されるため問題なし |
